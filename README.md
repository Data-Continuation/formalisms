# Data Continuation Formalism

Data is not governed because it exists. Data is governed when it seeks continuation into consequence.

This repository defines the Data Continuation Formalism (DCF): a formal ontology for distinguishing representational data from consequence-binding data.

## Core claim

A datum may be stored, generated, interpreted, predicted, or replayed without becoming reality-binding.

A datum becomes governance-relevant when it is routed toward continuation into consequence.

## Canonical boundary

```text
representation
→ proposed continuation
→ eligibility
→ inference window
→ commit
→ consequence
→ irreversibility
```

## Primary decision

```text
Can this data continue into reality in this role, through this transition, under this current state?
```

The answer is one of:

```text
ALLOW
DENY
FAIL_CLOSED
ALLOW_WITH_SIGNOFF
REDIRECT
ESCALATE
```

## Formal stack

```text
Data Continuation Formalism
  Defines representation-to-consequence continuation.

GCAT/BCAT
  Measures legitimacy capacity against consequence burden.

Transition Periodic Table
  Classifies the type of continuation crossing.

Inference Window
  Provides recursive pre-commit testing while data remains non-binding.

Rigel Recoverability
  Evaluates whether the post-transition state remains recoverable.

Receipt Chain
  Preserves the basis for the continuation decision.
```
